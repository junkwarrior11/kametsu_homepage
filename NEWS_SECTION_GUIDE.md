# 新着情報セクション - 運用ガイド

## 📰 概要

ホームページのトップに表示される「新着情報」セクションは、学校だよりと行事予定を自動的に統合して表示します。

## 🎯 表示仕様

### 掲載内容

1. **学校だより** (`newsletters` テーブル)
   - タイトル
   - 発行日
   - PDFファイルへのリンク
   - カテゴリ：📰 学校だより

2. **行事予定** (`events` テーブル)
   - 行事名
   - 対象年度・月
   - 説明文
   - PDFファイルへのリンク
   - カテゴリ：📅 行事予定

### 表示ルール

| 項目 | 設定値 | 説明 |
|------|--------|------|
| **表示期間** | 過去3ヶ月以内 | 古い情報は自動的に非表示になります |
| **表示件数** | 最新5件 | 学校だより + 行事予定を合わせて5件 |
| **ソート順** | 日付降順 | 新しい情報が上に表示されます |

### 動作イメージ

```

───────────────────────────────────────
📰 2026年度 学校だより 第12号
   2026-03-15  [PDFを見る]

📅 2026年度 卒業式
   2026-03-20  [PDFを見る]

📰 2026年度 学校だより 第11号
   2026-02-28  [PDFを見る]

📅 2026年度 修了式
   2026-03-24  [PDFを見る]

📰 2026年度 学校だより 第10号
   2026-02-15  [PDFを見る]
```

## 🔧 管理方法

### 新しい情報を追加する

1. **学校だよりをアップロード**
   - https://kametsu-homepage.netlify.app/admin-newsletter.html
   - タイトル、発行日、PDFファイルを入力
   - 「アップロード」ボタンをクリック
   - → **自動的に新着情報に表示されます**

2. **行事予定をアップロード**
   - https://kametsu-homepage.netlify.app/admin-events.html
   - タイトル、対象年度・月、PDFファイルを入力
   - 「アップロード」ボタンをクリック
   - → **自動的に新着情報に表示されます**

### 情報を削除する

1. **管理画面で削除**
   - 学校だより管理画面または行事予定管理画面
   - 削除したい項目の「削除」ボタンをクリック
   - → **新着情報からも自動的に削除されます**

2. **自動削除（3ヶ月経過後）**
   - 発行日/対象年月から3ヶ月が経過すると自動的に非表示
   - データベースからは削除されません（必要に応じて手動削除可能）

## 📅 期間フィルターの仕組み

### 現在日から3ヶ月前までの情報を表示

**例**：今日が 2026年3月15日 の場合

```javascript
3ヶ月前 = 2025年12月15日

表示される情報：
✅ 2026-03-15 の学校だより  → 表示
✅ 2026-02-28 の行事予定    → 表示
✅ 2026-01-20 の学校だより  → 表示
✅ 2025-12-20 の行事予定    → 表示
❌ 2025-12-10 の学校だより  → 非表示（3ヶ月以前）
❌ 2025-11-30 の行事予定    → 非表示（3ヶ月以前）
```

### なぜ3ヶ月？

- ✅ 春休み・夏休み・冬休み期間でも情報が残る
- ✅ 年4回程度の更新でも十分な情報量
- ✅ 保護者が過去の情報を確認できる期間

## 🔄 更新頻度の目安

### 推奨される更新頻度

| コンテンツ | 推奨頻度 | 理由 |
|-----------|---------|------|
| **学校だより** | 月1-2回 | 保護者への定期的な情報提供 |
| **行事予定** | 月1回 | 年間行事予定の更新 |

### 最低限の更新頻度

- **最低**: 3ヶ月に1回
- **理由**: 3ヶ月間更新がないと新着情報が空になる

## 💡 運用のヒント

### 情報が表示されない場合

1. **日付が正しいか確認**
   - 発行日が未来の日付になっていないか
   - 発行日が3ヶ月以上前になっていないか

2. **PDFがアップロードされているか確認**
   - 管理画面で「表示」ボタンが機能するか

3. **ブラウザのキャッシュをクリア**
   - `Ctrl + Shift + R` (Windows) / `Cmd + Shift + R` (Mac)

### 5件以上表示したい場合

**編集ファイル**: `/home/user/webapp/public/static/js/home.js`

```javascript
// 行207-208: 取得件数を変更
fetch('/api/tables/newsletters?limit=10&sort=-issue_date'),  // 10 → 20
fetch('/api/tables/events?limit=10&sort=-event_date')        // 10 → 20

// 行245: 表示件数を変更
const latestItems = recentItems.slice(0, 5);  // 5 → 10
```

### 期間フィルターを変更したい場合

**編集ファイル**: `/home/user/webapp/public/static/js/home.js`

```javascript
// 行229: 期間を変更
const threeMonthsAgo = new Date();
threeMonthsAgo.setMonth(threeMonthsAgo.getMonth() - 3);  // 3 → 6 (6ヶ月に変更)
```

## 📊 技術仕様

### データ取得フロー

```
1. データベースから取得
   ├─ newsletters テーブル (limit: 10, sort: -issue_date)
   └─ events テーブル (limit: 10, sort: -event_date)
   
2. 統合・変換
   ├─ カテゴリ名を追加 (学校だより / 行事予定)
   ├─ アイコンを設定 (fa-newspaper / fa-calendar)
   └─ 日付フィールドを統一 (issue_date / event_date → date)
   
3. フィルタリング
   └─ 3ヶ月以内の情報のみ抽出
   
4. ソート
   └─ 日付降順 (新しい順)
   
5. 表示
   └─ 最新5件のみ表示
```

### APIエンドポイント

```javascript
// 学校だより取得
GET /api/tables/newsletters?limit=10&sort=-issue_date

// 行事予定取得
GET /api/tables/events?limit=10&sort=-event_date
```
